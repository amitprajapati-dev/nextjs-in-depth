# Next.js Fundamentals Interview Questions

## 📖 Introduction

This document contains the most common **Next.js Fundamentals** interview questions.

These questions cover the concepts discussed in this section and are frequently asked in internships, fresher interviews, and junior developer roles.

---

# 1. What is Next.js?

**Answer:**

Next.js is an open-source React framework developed by Vercel. It helps developers build fast, scalable, and production-ready web applications by providing built-in features such as file-based routing, server-side rendering, static site generation, API routes, image optimization, and more.

---

# 2. Is Next.js a Library or a Framework?

**Answer:**

Next.js is a **React Framework**.

React focuses on building user interfaces, while Next.js provides additional tools required to build complete web applications.

---

# 3. Who Developed Next.js?

**Answer:**

Next.js was developed and is maintained by **Vercel**.

---

# 4. Why was Next.js Created?

**Answer:**

React does not provide features like routing, SEO, server-side rendering, static site generation, or API routes by default.

Next.js solves these problems by providing them out of the box.

---

# 5. Why do companies prefer Next.js?

**Answer:**

Companies use Next.js because it offers:

- Better Performance
- SEO Optimization
- Faster Loading
- Built-in Routing
- Improved Developer Experience
- Scalability
- Production-ready Features

---

# 6. What is File-Based Routing?

**Answer:**

File-Based Routing means routes are automatically created based on the folder and `page.tsx` file structure inside the `app` directory.

No manual route configuration is required.

---

# 7. What is the App Router?

**Answer:**

The App Router is the modern routing system introduced in Next.js 13.

It is based on the `app` directory and supports features like Server Components, nested layouts, loading UI, error handling, and streaming.

---

# 8. What is the difference between React and Next.js?

| React | Next.js |
|--------|----------|
| UI Library | React Framework |
| Manual Routing | Built-in Routing |
| No Built-in SEO | Excellent SEO Support |
| Client-side by Default | Multiple Rendering Methods |
| UI Focused | Full-stack Capabilities |

---

# 9. What is Rendering?

**Answer:**

Rendering is the process of converting React components into HTML that browsers can display.

---

# 10. What rendering methods does Next.js support?

**Answer:**

Next.js supports:

- Client-Side Rendering (CSR)
- Server-Side Rendering (SSR)
- Static Site Generation (SSG)
- Incremental Static Regeneration (ISR)
- Partial Prerendering (PPR)

---

# 11. What is Hydration?

**Answer:**

Hydration is the process where React attaches JavaScript functionality to the HTML generated by the server, making the page interactive.

---

# 12. What is the purpose of the `app` folder?

**Answer:**

The `app` folder contains the application's pages, layouts, routes, loading UI, error pages, and other core application files.

It is the heart of a modern Next.js project.

---

# 13. What is the purpose of the `public` folder?

**Answer:**

The `public` folder stores static assets such as:

- Images
- Icons
- Videos
- PDFs
- Favicons

These files can be accessed directly using their URL path.

---

# 14. What is `package.json`?

**Answer:**

`package.json` stores project metadata, dependencies, scripts, and configuration.

It is used by npm to manage the project.

---

# 15. Why should `node_modules` not be pushed to GitHub?

**Answer:**

Because it contains installed packages that can be regenerated using:

```bash
npm install
```

Uploading it makes the repository unnecessarily large.

---

# 16. What is the purpose of `next.config.ts`?

**Answer:**

It is the main configuration file for Next.js.

It is used to customize framework behavior such as image domains, redirects, headers, and experimental features.

---

# 17. What are Server Components?

**Answer:**

Server Components run on the server by default.

They reduce the amount of JavaScript sent to the browser, resulting in better performance and improved security.

---

# 18. When should you use a Client Component?

**Answer:**

Use a Client Component only when you need:

- useState
- useEffect
- Event Handlers
- Browser APIs

Otherwise, prefer Server Components.

---

# 19. What are some key features of Next.js?

**Answer:**

- File-Based Routing
- Server Components
- Client Components
- SSR
- SSG
- ISR
- API Routes
- Middleware
- Image Optimization
- Font Optimization
- Metadata API
- Fast Refresh

---

# 20. Which router should be used for new projects?

**Answer:**

The **App Router** should be used for all new Next.js projects because it is the recommended routing system and supports modern features like Server Components, nested layouts, loading UI, and streaming.

---
