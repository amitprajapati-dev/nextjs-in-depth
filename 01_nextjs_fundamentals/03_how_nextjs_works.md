# How Next.js Works

## 📖 Introduction

Before writing Next.js applications, it is important to understand what happens behind the scenes when a user visits a website.

Understanding this flow will help you learn topics like Routing, Rendering, Data Fetching, Server Components, and SEO much more easily.

Think of Next.js as a bridge between the browser and your application.

---

# 🌐 High-Level Workflow


This entire process happens within milliseconds.

---

# Step 1 — User Sends a Request

Imagine a user types:

```

https://example.com/about

```

or clicks a link inside your website.

The browser sends a request to the Next.js server asking for the `/about` page.

---

# Step 2 — Next.js Receives the Request

The Next.js server receives the request.

Now it decides:

- Which page is requested?
- Does the page already exist?
- Should it generate HTML?
- Should it fetch data?
- Which rendering strategy should be used?

This decision-making process is one of the biggest strengths of Next.js.

---

# Step 3 — React Components Are Rendered

Once the correct page is found, Next.js renders the React components.

For example:

```jsx
export default function About() {
    return <h1>About Us</h1>;
}
```

React creates the UI for the requested page.

At this stage, the page still isn't visible to the user.

---

# Step 4 — HTML is Generated

Next.js converts the React components into HTML.

Example:

React Component

```jsx
<h1>About Us</h1>
```

Generated HTML

```html
<h1>About Us</h1>
```

Browsers understand HTML much faster than JavaScript.

That's why users see content quickly.

This is one of the reasons Next.js performs better than many traditional React applications.

---

# Step 5 — HTML is Sent to the Browser

The generated HTML is sent back to the user's browser.

At this moment:

✅ The page is visible.

❌ But it is not fully interactive yet.

Buttons, forms, and event handlers are still inactive.

---

# Step 6 — Hydration

After displaying the HTML, the browser downloads the JavaScript files.

React then connects JavaScript to the existing HTML.

This process is called **Hydration**.

Hydration makes the website interactive.

For example:

Before Hydration:

- Buttons don't respond.
- Forms don't work.
- Click events don't execute.

After Hydration:

- Buttons become clickable.
- Forms work.
- Navigation becomes smooth.
- The application becomes fully interactive.

---

# 🧠 What is Hydration?

Hydration is the process where React attaches JavaScript behavior to the HTML generated by the server.

In simple words:

> **HTML makes the page visible.**
>
> **JavaScript makes the page interactive.**

---

# 🌍 Real-World Example

Imagine ordering food online.

The restaurant first prepares your meal.

Only after the meal is ready does the delivery partner bring it to you.

Similarly,

Next.js first prepares the page (HTML), then JavaScript arrives to make everything interactive.

# 🎤 Interview Questions

### Q1. What happens when a user opens a Next.js website?

**Answer:**

The browser sends a request to the Next.js server. The server finds the requested page, renders the React components, generates HTML, and sends it back to the browser. After the HTML is displayed, JavaScript loads and React hydrates the page, making it fully interactive.

---

### Q2. What is Hydration?

**Answer:**

Hydration is the process where React attaches JavaScript functionality to the HTML generated by the server. This makes the page interactive without re-rendering it from scratch.

---

### Q3. Why does Next.js load faster than a traditional React application?

**Answer:**

Next.js can send pre-rendered HTML to the browser before JavaScript finishes loading. This allows users to see content quickly, improves SEO, and provides a better user experience.
